Depth Anything Project

From paper presentation to local teacher-student depth system.

This version is no longer just a tiny supervised fine-tune. It now follows the original paper logic more closely: keep the architecture fixed, expand the data with pseudo labels, and train a student on more images than the labeled seed set.

What changed

  • Added a `2000` image indoor pseudo-label pool
  • Generated teacher pseudo-depth maps
  • Trained a student on `450` labeled plus `2000` pseudo-labeled images
  • Measured all three runs on the same held-out test split
Paper Claim

Data scale is the story.

The original paper’s headline is not a radically new architecture. It is a strong depth model plus much more training data, including pseudo-labeled data from a teacher-student pipeline.

63.5M

The original recipe is huge. This project uses a local fraction of that idea: small labeled seed set, larger pseudo-labeled indoor pool, same core model family.

Architecture

What stayed fixed.

  • Pretrained model: LiheYoung/depth-anything-small-hf
  • Architecture family: Depth Anything Small / DPT-style monocular depth estimation
  • Input: single RGB image
  • Output: dense depth map
Important distinction
  • The architecture did not change
  • The main experiment change was data scale through pseudo labels
  • This is best interpreted as relative depth estimation, not exact metric distance from one image
Dataset Growth

Start small, then add more.

450 labeled seed 2000 pseudo-labeled indoor 2450 student total 59-image held-out test

This is the closest local version of the original paper story: the student trains on materially more data than the initial labeled set.

Teacher-Student Pipeline

Actual local workflow.

  1. Take the pretrained Depth Anything Small model as the teacher
  2. Export `2000` indoor RGB images into an unlabeled pool
  3. Run the teacher on that pool and save pseudo-depth maps
  4. Train the student on `450` labeled NYU-v2 images plus `2000` pseudo-labeled images
  5. Evaluate teacher, supervised-only student, and teacher-student student on the same test split

Why this matters

  • More faithful to the original paper logic
  • No architecture drift
  • Lets the data story carry the project
Measured Results

Three-way comparison.

Teacher: 0.1467 / 0.8381 Supervised: 0.1810 / 0.7163 Teacher-student: 0.1682 / 0.7531
  • Teacher-student improved over the supervised-only student
  • The zero-shot teacher still remained strongest
  • The extra pseudo-labeled data helped, but not enough to surpass the teacher

Conclusion

The project now matches the original story much more closely.

The final honest claim is not that the local student beat the teacher. It is that the repo now demonstrates the teacher-student pseudo-label expansion idea, and that the added data measurably improved the student over the tiny supervised-only baseline.
