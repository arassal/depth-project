Why Depth

Monocular depth is useful and fundamentally ambiguous.

One RGB image can support strong relative depth structure, but not guaranteed absolute metric depth. That is why alignment and relative-depth training matter.

Use cases from the original presentation

  • AR occlusion and 3D placement
  • Portrait effects and relighting
  • Navigation and robotics perception
  • 3D reconstruction from ordinary photos
Paper Claim

Data scale, not a new architecture, is the headline.

The original paper uses a pretrained visual backbone, a familiar depth head, and a teacher-student pseudo-label pipeline. The main claim is that scaling data quality and data quantity pushes zero-shot generalization.

63.5M

Total images in the paper’s large-scale recipe. The local project does not fake this scale; it builds the same story in a realistic small-system form.

Engineering Translation

What was preserved from the slides.

  • Depth Anything Small as the default model
  • NYU-v2 as the benchmark dataset
  • Affine-invariant training logic
  • Aligned AbsRel and delta1 evaluation
  • A later pseudo-label self-training stage
What was intentionally cut
  • No fake reproduction of full 63.5M-image training
  • No oversized architecture changes
  • No metric-depth claims beyond what the pipeline supports
  • No results invented before the dataset and run complete

Dataset Choice

NYU-v2 is the correct first benchmark.

It matches the original presentation, fits the indoor-depth story, and is realistic for a local fine-tuning and evaluation run.

Execution Flow

Baseline run, then evaluation, then plots.

  1. Prepare NYU-v2 into train, val, and test manifests
  2. Fine-tune Depth Anything Small on NYU-v2
  3. Evaluate with aligned AbsRel and delta1
  4. Generate qualitative panels and metric plots
  5. Optionally run pseudo-label self-training

Expected outputs

  • Best checkpoint
  • Training history JSONL
  • Test summary JSON
  • Per-image metric CSV
  • Qualitative prediction panels
  • Loss and metric plots
Results Section

The pipeline is ready for the results that belong here.

The proof-of-concept run is complete on a clean NYU-v2 subset. The key result is that the small supervised fine-tune improved validation but hurt held-out test performance, which is useful evidence of overfitting on a small split.

Zero-shot AbsRel 0.1467 Zero-shot delta1 0.8381 Fine-tuned AbsRel 0.1810 Fine-tuned delta1 0.7163 59 before/after panels

What was reported

  • Zero-shot baseline before fine-tuning
  • Two-epoch supervised fine-tune on 450 train images
  • Validation improved from epoch 1 to epoch 2
  • Held-out test performance degraded, indicating overfitting on the PoC split
