This project now does two things beyond the original small proof of concept: it follows the paper-style teacher-student data expansion, and it adds a lightweight RGB-guided residual refinement head on top of the student’s coarse depth prediction.

Student architecture modification

Refinement head

The student no longer outputs the backbone depth map directly. It now predicts a correction using `RGB + coarse depth`, then adds that residual back to the coarse map.

  • Base model: `Depth Anything Small`
  • New module: `3` convolution layers with `GELU`
  • Input to new head: `RGB + coarse depth`
  • Output: residual correction map
  • Final depth: `coarse + scaled residual`

Data growth and pseudo-label expansion

How to read the model outputs

Each NYU-v2 panel is organized as RGB | GT | Pred | Error. The key question is whether the model preserves large scene structure and relative depth ordering. The system should be interpreted as relative depth estimation, not exact metric distance measurement from one image.

Exact refined workflow

cd /home/alexander/depth-project
source .venv/bin/activate
python src/prepare_indoor_pseudo_pool.py \
  --root data/unlabeled/indoor_teacher_pool \
  --limit 2000

python src/pseudo_label.py --config configs/teacher_student_poc.yaml
python src/train.py --config configs/teacher_student_refined.yaml
python src/eval.py --config configs/teacher_student_refined.yaml \
  --checkpoint checkpoints/teacher_student_refined_best.pt
