Main conclusion

The fine-tuned checkpoint performed worse than the zero-shot baseline on the held-out test subset. That makes this run a valid proof-of-concept and a useful hyperparameter-analysis artifact, not a fake success story.

  • validation still improved across epochs
  • test performance dropped after fine-tuning
  • the result suggests overfitting on a small training subset
  • all 59 before/after image panels were saved for inspection

How to read the NYU-v2 panels

Each saved panel is organized as RGB | GT | Pred | Error. The main question is whether the model preserves the large scene layout and the near-versus-far ordering, not whether it produces exact real-world distance in meters from a single photo.

Second dataset: KITTI Tiny

KITTI Tiny is included as a zero-shot outdoor driving demo to show that the pretrained model is not locked to indoor scenes.

What this demonstrates

  • The pretrained model generalizes to an outdoor scene without extra tuning
  • The output is most trustworthy as relative depth structure
  • The second dataset is useful because it shows indoor and outdoor behavior side by side

Interactive Upload Demo

You can run a live local web app that accepts any uploaded image and generates a depth map with the pretrained model.

cd /home/alexander/depth-project
source .venv/bin/activate
python src/demo_web_app.py

Then open http://127.0.0.1:8000 in the browser.

What to expect

  • The upload demo runs zero-shot inference, not the overfit fine-tuned checkpoint
  • It is best at relative depth structure, not perfect metric depth
  • Uploaded inputs and generated depth images are saved under outputs/web_demo/

This page is intentionally honest: the zero-shot baseline beat the small fine-tuned model on the held-out subset, and that outcome is exactly the kind of result a credible proof-of-concept presentation should surface.
